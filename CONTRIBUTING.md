# Contributing to Exploidus

Thank you for your interest in contributing to Exploidus.

Exploidus is an open-source operating system built from scratch. Every contribution, whether code, documentation, testing, or bug reports, is appreciated.

## Getting Started

1. Fork the repository.
2. Clone your fork.

```bash
git clone https://github.com/your-username/Exploidus.git
cd Exploidus
```

3. Create a new branch.

```bash
git checkout -b feature/my-feature
```

4. Make your changes.
5. Commit your changes.

```bash
git commit -m "Add scheduler improvement"
```

6. Push your branch.

```bash
git push origin feature/my-feature
```

7. Open a Pull Request.

---

## Coding Guidelines

- Write clean and readable code.
- Keep functions focused and small.
- Comment complex logic when necessary.
- Follow the existing code style.
- Avoid unnecessary dependencies.

---

## Commit Messages

Examples:

```
Fix page allocator bug
Implement APIC timer
Improve memory manager
Update documentation
```

---

## Reporting Bugs

Please include:

- Architecture
- Build environment
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots/logs (if available)

---

## Feature Requests

Before requesting a feature:

- Check existing Issues.
- Explain why the feature is useful.
- Keep proposals focused.
