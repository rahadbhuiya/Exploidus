# Security Policy

## Supported Versions

Only the latest development branch receives security fixes.

| Version | Supported |
|----------|-----------|
| Latest   | Done |
| Older    | Cancel |

---

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead:

- Open a private GitHub Security Advisory (if enabled), or
- Contact the maintainer directly.

Include:

- Description
- Impact
- Steps to reproduce
- Possible fix (optional)

Security reports will be reviewed as soon as possible.

Thank you for helping keep Exploidus secure.
